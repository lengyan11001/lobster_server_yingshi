# 接口鉴权规范

后续新增或修改接口时，必须按以下规则做好鉴权。

## 一、需登录的接口（默认）

所有**读写用户数据、计费、技能、发布、设置、素材、对话**等接口，必须校验用户身份：

- 使用 `current_user: User = Depends(get_current_user)`，保证只有携带有效 JWT（`Authorization: Bearer <token>`）的请求才能访问。
- 业务逻辑中按 `current_user.id` 做数据隔离（只查/改当前用户的资源）。

**示例**：

```python
from .auth import get_current_user
from ..models import User

@router.get("/api/xxx")
def get_xxx(current_user: User = Depends(get_current_user), db: Session = Depends(get_db)):
    # 只返回 current_user 的数据
    ...
```

## 二、可不鉴权的接口（例外，需明确理由）

仅以下情况可不要求登录：

| 类型 | 示例 | 说明 |
|------|------|------|
| 登录/注册/验证码 | `/auth/login`, `/auth/register-phone`, `/auth/sms/send`, `/auth/captcha`（`/auth/register` 已关闭） | 未登录前必须可访问 |
| 登录回调 | `/auth/wechat-callback`, `/auth/sutui-callback` | 第三方重定向，无法带 JWT |
| 公开配置 | `/api/edition` | 前端需在登录前获知版本/能力，用于展示登录方式 |
| 健康检查 | `/api/health`, `/api/lan-ip` | 监控/内网发现用 |
| 支付/消息回调 | `/api/recharge/fubei-notify`, `/api/wechat`, `/api/wecom/callback/...`, `/api/messenger/callback/...` | 由第三方服务器调用，用签名/密钥校验 |
| 管理员接口 | 如 `/api/recharge/complete` | 用 `X-Admin-Secret` 等独立鉴权 |

新增「无需登录」的接口时，应在代码注释或本表中说明原因，避免误开敞口。

## 三、资源归属校验

- 对「按 ID 操作」的接口（如 `/api/accounts/{account_id}`），除通过 `get_current_user` 鉴权外，必须校验 `account_id` 属于 `current_user`（或该资源表有 `user_id` 且等于 `current_user.id`），防止越权访问。

## 四、清单（新增接口时自检）

- [ ] 该接口是否涉及用户数据/计费/技能/发布/设置/素材/对话？→ 是则必须 `Depends(get_current_user)`
- [ ] 若为「无需登录」的例外，是否已写明原因？
- [ ] 若有资源 ID 参数，是否校验归属当前用户？

---

## 五、技能使用限制（付费解锁）

- 在 **skill_registry.json** 中为技能包设置 `unlock_price_yuan`（如 `200`），即表示「需支付 200 元解锁后才能使用」。
- 用户未解锁时：**GET /capabilities/available** 不会返回该技能下的能力；**POST /capabilities/pre-deduct**、**POST /capabilities/record-call** 会返回 403「该能力属于付费技能，请先在技能商店付费解锁后再使用」。
- 用户通过 **技能商店 → 创建解锁订单 → 支付 → 管理员 complete 或微信回调** 完成解锁后，即可正常使用该技能下的能力。
