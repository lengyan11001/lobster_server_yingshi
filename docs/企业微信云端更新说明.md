# 企业微信（云端 lobster-server）更新说明

## 1. 数据模型

- **WecomPendingMessage（待处理消息表）**：存储企微回调解密后的消息，供本地轮询拉取。字段：wecom_config_id、from_user、to_user、content、msg_type、status（pending/replied/failed）、reply_text、created_at。
- **WecomConfig**：增加 **secret** 字段，用于获取 access_token 并调用企微「发送应用消息」接口（轮询模式下由云端代发回复）。

## 2. 迁移

- 启动时自动执行 **\_migrate_wecom_config_secret()**：为已有 `wecom_configs` 表补 `secret` 列（若不存在）。

## 3. 回调（POST）

- **流程**：解密 → 写入待处理队列（WecomPendingMessage）→ **立即返回占位回复**（“收到，正在处理。”）。
- **不再在云端调 AI**；真实回复由本地轮询拉取后生成，再通过 **submit-reply** 由云端代发应用消息。

## 4. 轮询接口（需鉴权）

- **GET /api/wecom/pending**  
  - 拉取待处理消息。  
  - 查询参数：`callback_path`（可选）、`limit`（默认 20，最大 100）。  
  - 鉴权：请求头 **X-Forward-Secret** 与配置项 **wecom_forward_secret** 一致时通过；未配置则不做校验。

- **POST /api/wecom/submit-reply**  
  - 提交 `message_id` + `reply_text`。  
  - 云端用该消息所属应用的 **corp_id、secret** 取 access_token，再调企微「发送应用消息」接口把回复推给用户。  
  - 鉴权：同上，X-Forward-Secret 与 wecom_forward_secret 一致。

## 5. 配置与前端

- **WecomConfig CRUD**：支持填写、展示「应用 Secret」；列表展示 **Secret: 已配置 / 未配置**（不展示明文）。
- **配置项 wecom_forward_secret**（可选）：与本地轮询时请求头 **X-Forward-Secret** 一致；不设则轮询接口不做鉴权（仅内网或隧道时建议设置）。

## 6. 本地使用方式

- 轮询拉取：`GET https://<云端>/api/wecom/pending`，请求头带 `X-Forward-Secret: <wecom_forward_secret>`。
- 提交回复：`POST https://<云端>/api/wecom/submit-reply`，Body `{"message_id": 1, "reply_text": "回复内容"}`, 请求头同样带 `X-Forward-Secret`。
- **wecom_forward_secret** 已由部署时生成并写入服务器 .env，本地配置同一密钥即可（见下发给你的密钥）。
