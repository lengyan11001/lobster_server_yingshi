## 📋 参数格式总结

### duration 参数格式
| 模型系列 | 格式 | 示例 | 默认值 |
|---------|------|------|--------|
| super-seed2 | 整数 | `5`, `10` | `5` |
| wan/v2.6 | 字符串 | `"5"`, `"10"` | `"5"` |
| vidu | 整数 | `5`, `10` | `5` |
| seedance v1/v1.5 | 字符串 | `"5"`, `"10"` | `"5"` |
| sora-2 | 整数 | `5`, `10` | `5` |
| kling | 整数 | `5`, `10` | `5` |
| veo-3.1 | 字符串（特殊） | `'4s'`, `'6s'`, `'8s'` | `'6s'` |
| grok | 整数 | `5`, `10` | `5` |
| jimeng | 整数 | `5`, `10` | `5` |
| 其他 | 整数 | `5`, `10` | `5` |

### image_url 处理
- **图片模型**:
  - `flux-2/flash`, `nano-banana`: 使用 `image_urls` (数组)
  - `jimeng-*`: 使用 `image_url` (字符串)
  - 其他: 使用 `image_url` (字符串)

- **视频模型**:
  - `super-seed2`: 使用 `filePaths` (数组)
  - 其他: 使用 `image_url` (字符串)

### aspect_ratio 处理
- **图片模型**: `nano-banana` 使用 `aspect_ratio` (默认 "1:1")
- **视频模型**: 大部分模型支持 `aspect_ratio` (默认 "16:9")
- **特殊**: `super-seed2` 使用 `ratio` 而非 `aspect_ratio`

## 🔍 潜在问题检查

### 1. 通用处理逻辑中的 duration
**位置**: `_normalize_video_generate_payload` 的通用处理部分（第 1020-1022 行）

**当前代码**:
```python
# 如果没有 duration，添加默认值
if "duration" not in out and duration is not None:
    out["duration"] = duration
```

**问题**: `duration` 在函数开头被设置为整数 `5`（如果为 None），但某些模型可能需要字符串格式。

**建议**: 通用处理中，如果用户没有传入 `duration`，不添加默认值，让上游 API 处理。或者根据模型类型智能判断。

**当前状态**: ⚠️ 需要确认上游 API 是否接受整数格式的 duration
