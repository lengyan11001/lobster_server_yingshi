# 企业微信在服务器上的部署说明

当 **lobster_server** 部署在具有公网 IP 或域名的服务器上时，企业微信「接收消息」回调可以正常使用，企微服务器能访问你的回调 URL。

## 1. 配置公网根地址（必做）

在服务器上的 **.env** 中设置：

```bash
# 公网访问根地址，用于生成企业微信回调 URL 等（不要带末尾斜杠）
PUBLIC_BASE_URL=https://你的域名或IP
```

例如：

- `PUBLIC_BASE_URL=https://lobster.example.com`
- `PUBLIC_BASE_URL=https://123.45.67.89`（若用 IP 且上了 HTTPS）

这样在「企业微信配置」页中展示的**回调 URL** 才是企微后台能访问的地址。若不设置，在反向代理后可能拿到内网地址（如 `http://127.0.0.1:8000`），填到企微会校验失败。

## 2. 网络与 HTTPS

- **公网可达**：企业微信服务器会主动请求你的回调 URL，服务器必须能从互联网被访问（公网 IP 或域名解析到该机）。
- **建议 HTTPS**：企业微信对回调地址推荐使用 HTTPS；若用 HTTP，部分环境可能有限制。

## 3. 反向代理（Nginx / Caddy 等）

若前面有 Nginx/Caddy 做反向代理和 HTTPS：

- 在 .env 中设置 **PUBLIC_BASE_URL** 为对外访问的完整根地址（如 `https://lobster.example.com`），即可保证回调 URL 正确。
- 代理需把请求正确转到后端（如 `http://127.0.0.1:8000`），无需在应用内再改端口。

## 4. 流程小结

1. 在服务器 .env 配置 **PUBLIC_BASE_URL**。
2. 启动/重启 lobster_server。
3. 登录后台 → 技能商店 → 企业微信自动回复 → **配置** → 添加配置（Token、EncodingAESKey 等）。
4. 复制配置页展示的**回调 URL**，填入企业微信管理后台「接收消息」的服务器配置。
5. 保存后企微会发 GET 校验；通过后即可接收消息并自动回复。
