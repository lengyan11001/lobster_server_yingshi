# 龙虾 (Lobster) 云服务器部署说明

在云服务器（Linux）上跑起在线版需要：**安装环境 → 部署包（lobster-server）→ 配置 .env → 启动后端与 MCP →（可选）反向代理与开机自启**。

**目录约定**

- **`lobster/`**：完整源码，本地开发、打 online 版、改功能都在此目录。
- **`lobster-server/`**：仅服务器部署用，由 `lobster/scripts/build_server_package.sh` 从 `lobster` 同步得到；云服 Git 拉取后只使用 **`lobster-server/`** 启动服务。

更新部署包（在开发机、于 `lobster` 根目录执行）：  
`./scripts/build_server_package.sh`，再提交并推送 `lobster-server/`。

---

## 一、服务器上要跑什么

| 进程 | 端口 | 说明 |
|------|------|------|
| **Backend** | 8000 | FastAPI 主服务：网页、API、登录、对话、计费、发布等 |
| **MCP** | 8001（仅本机） | 能力网关：速推/技能/发布等工具，只供 Backend 调用，不对外暴露 |
| **OpenClaw Gateway** | 18789（可选） | 若用 OpenClaw 做对话路由才需要，否则对话可走直连模型 |

对外只需开放 **8000**（或由 Nginx 反代后的 80/443）。8001、18789 仅本机访问。

---

## 二、环境要求

- **操作系统**：Linux（如 Ubuntu 22.04、CentOS 7+）
- **Python**：3.10 或 3.12，且能装 pip 依赖
- **Node.js**（可选）：仅在使用 OpenClaw Gateway 时需要
- **Chromium/Playwright**（可选）：仅在使用「发布到抖音」等浏览器能力时需要

```bash
# Ubuntu/Debian 示例
sudo apt update
sudo apt install -y python3.10 python3.10-venv python3-pip

# 若需要发布到抖音（无头浏览器）
# sudo apt install -y libnss3 libnspr4 libatk1.0-0 libatk-bridge2.0-0 libcups2 libdrm2 libxkbcommon0 libxcomposite1 libxdamage1 libxfixes3 libxrandr2 libgbm1 libasound2
# 安装后进入项目目录执行: python3 -m playwright install chromium
```

## 三、部署步骤

### 0. Git 克隆后快速启动（推荐）

代码已推送到 Git 仓库（且含 `lobster-server/`）时，在服务器上**进入部署包目录**执行：

```bash
git clone <你的仓库地址> /opt/repo
cd /opt/repo/lobster-server
chmod +x scripts/*.sh
./scripts/server_install.sh
```

- 若没有 `.env`，脚本会复制 `.env.example` 并提示你先编辑再启动。
- 编辑 `.env` 填写 `SECRET_KEY`、`SUTUI_SERVER_TOKEN` 等（见下文「3. 配置 .env」）。

然后一键启动 MCP + Backend：

```bash
./scripts/server_start.sh
```

- 会先检测 8001 是否已有 MCP，无则后台启动 MCP，再前台启动 Backend（端口 8000）。
- 访问：`http://服务器IP:8000`。生产环境建议改用 systemd（见下文「方式 B」）。

### 1. 上传代码（非 Git 时）

将 **`lobster-server/`** 整个目录上传到服务器（例如 `/opt/lobster-server`），保证目录内包含：

- `backend/`、`mcp/`、`static/`、`publisher/`、`openclaw/`（至少 `openclaw.json`）
- `scripts/`（含 `server_install.sh`、`server_start.sh`）
- `requirements.txt`、`.env.example`、`skill_registry.json`

### 2. 安装 Python 依赖（未用 server_install.sh 时）

```bash
cd /opt/lobster-server
python3 -m venv .venv
source .venv/bin/activate   # Linux/Mac
pip install -r requirements.txt
```

### 3. 配置 .env

```bash
cd /opt/lobster-server
cp .env.example .env
vim .env   # 或 nano .env
```

**必填 / 强烈建议：**

| 变量 | 说明 |
|------|------|
| `SECRET_KEY` | 随机长字符串，用于 JWT 等，生产环境必改 |
| `SUTUI_SERVER_TOKEN` | 速推 API Token，能力由服务器转发时必填 |
| `HOST` | 填 `0.0.0.0`，以便外网访问 |
| `PORT` | 默认 8000 |
| `LOBSTER_RECHARGE_ADMIN_SECRET` | 完成充值/技能解锁订单时用的密钥，仅服务端知晓 |

**按需填写：**

- `PUBLIC_BASE_URL`：图生视频等需被速推拉图时，填公网访问地址（如 `https://your-domain.com`）
- `DEBUG`：生产建议 `false`
- `LOG_LEVEL`：生产可设 `info` 减少日志

**MCP 读到的环境**：Backend 和 MCP 都会读部署根目录（即 `lobster-server` 根目录）下的 `.env`；若用 systemd，工作目录设为该根目录，或把 `SUTUI_SERVER_TOKEN` 等写进 systemd 的 `Environment=`。

### 4. 启动方式

**方式 A：前台直接跑（适合先试跑）**

```bash
cd /opt/lobster-server
source .venv/bin/activate
export PYTHONPATH=/opt/lobster-server

# 终端 1：MCP（必须先起，否则 Backend 会尝试自启一次）
python3 -m mcp --port 8001

# 终端 2：Backend
python3 -m backend.run
```

然后访问 `http://服务器IP:8000`。

**方式 B：用 systemd 常驻（推荐生产）**

以下路径均以部署根目录 `/opt/lobster-server` 为例，先建两个 service 文件。

`/etc/systemd/system/lobster-mcp.service`：

```ini
[Unit]
Description=Lobster MCP Server
After=network.target

[Service]
Type=simple
User=www-data
WorkingDirectory=/opt/lobster-server
Environment=PYTHONPATH=/opt/lobster-server
EnvironmentFile=/opt/lobster-server/.env
ExecStart=/opt/lobster-server/.venv/bin/python3 -m mcp --port 8001
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

`/etc/systemd/system/lobster-backend.service`：

```ini
[Unit]
Description=Lobster Backend
After=network.target lobster-mcp.service

[Service]
Type=simple
User=www-data
WorkingDirectory=/opt/lobster-server
Environment=PYTHONPATH=/opt/lobster-server
EnvironmentFile=/opt/lobster-server/.env
ExecStart=/opt/lobster-server/.venv/bin/python3 -m backend.run
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

启用并启动：

```bash
sudo systemctl daemon-reload
sudo systemctl enable lobster-mcp lobster-backend
sudo systemctl start lobster-mcp lobster-backend
sudo systemctl status lobster-mcp lobster-backend
```

日志：

```bash
journalctl -u lobster-backend -f
journalctl -u lobster-mcp -f
```

## 四、可选：Nginx 反代 + HTTPS

对外只暴露 80/443，由 Nginx 转发到本机 8000：

```nginx
server {
    listen 80;
    server_name your-domain.com;
    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

HTTPS 可用 `certbot` 申请证书并改 `listen 443 ssl` 等。

**.env 中**：若用户通过 `https://your-domain.com` 访问，则 `PUBLIC_BASE_URL=https://your-domain.com`（图生视频等会用到）。

## 五、可选：OpenClaw Gateway

若希望对话走 OpenClaw（多模型路由），需再起 OpenClaw Gateway（Node）：

- 安装 Node.js，在部署根目录（如 `/opt/lobster-server`）执行 `npm install openclaw`（或使用已有 nodejs 目录）
- 启动：`OPENCLAW_CONFIG_PATH=/opt/lobster-server/openclaw/openclaw.json OPENCLAW_STATE_DIR=/opt/lobster-server/openclaw node node_modules/openclaw/openclaw.mjs gateway --port 18789`
- `.env` 中配置 `OPENCLAW_GATEWAY_URL=http://127.0.0.1:18789` 和 `OPENCLAW_GATEWAY_TOKEN`

不配 OpenClaw 时，对话仍可用直连模型（在 openclaw/.env 或系统配置里填各模型 API Key）。

---

## 六、可选：发布到抖音（Playwright Chromium）

发布到抖音依赖无头浏览器，需安装 Playwright 的 Chromium：

```bash
cd /opt/lobster-server
source .venv/bin/activate
python3 -m playwright install chromium
```

若报缺系统库，按 Playwright 文档安装对应依赖（如 `libnss3`、`libatk-bridge2.0-0` 等）。

## 七、部署自检清单

- [ ] Python 3.10+ 与 `pip install -r requirements.txt` 已执行
- [ ] `.env` 已从 `.env.example` 复制并填写 `SECRET_KEY`、`SUTUI_SERVER_TOKEN`
- [ ] MCP 已启动且监听 8001（本机）
- [ ] Backend 已启动且监听 8000（0.0.0.0）
- [ ] 浏览器访问 `http://服务器IP:8000` 能打开登录页
- [ ] 能注册/登录，且对话里能力（如图片生成）可用（依赖 SUTUI_SERVER_TOKEN）
- [ ] 若用抖音发布：已安装 Playwright Chromium 且 `browser_data` 等目录可写
- [ ] 生产环境：已设 `DEBUG=false`，已配置 Nginx + HTTPS，已设 `LOBSTER_RECHARGE_ADMIN_SECRET`

## 八、常见问题

- **能力调用报「Token 未配置」**：检查 `.env` 中 `SUTUI_SERVER_TOKEN` 是否填写，且 MCP 进程能读到该环境变量（同一 `.env` 或 systemd 的 EnvironmentFile）。
- **图生视频报 Failed to download**：速推要能访问素材 URL，请把 `PUBLIC_BASE_URL` 设为公网可访问的根地址（如 `https://your-domain.com`）。
- **发布到抖音失败**：确认已 `playwright install chromium`，且运行用户有写 `browser_data` 的权限。

按上述步骤即可在云服务器上跑起龙虾在线版。
